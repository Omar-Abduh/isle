## Rule

When a task involves a specific API, library, framework, or tool — especially one that may have changed — **fetch the official docs first** before writing code or making assumptions.

## When in Doubt, Fetch Docs

If you're unsure about:
- API method signatures or options
- Configuration schemas
- Version-specific behavior
- Migration paths
- Best practices

Fetch the official docs or the library's README on GitHub before proceeding.
